## 什么是 AutoCode

AutoCode 是 Claude Code 的插件。安装后，Claude Code 变成一台全自动编程机器。

你说一句话，它自动完成：选技术方案、搭框架、写代码、写测试、查安全、查性能、修bug、保存进度、从头复盘、部署上线。

**整个过程你不需要懂任何编程知识。** 你甚至不需要知道"编程"是什么。

它问你"用什么语言开发？"——你回答"你看着办"——它自动选运行最快、最省钱、最稳定的方案，然后告诉你："我选了一个方案，同样一台6块钱的服务器，别人扛100人，你的能扛1万人。"

**用得越久，它越懂你。** 你的偏好、你的风格、你喜欢什么不喜欢什么，它全部记住。到第20次用的时候，它几乎不问你任何问题——直接按你的习惯做，做完你一看，刚好就是你要的。

## 🌍 和目前的AI编程工具有什么不同

目前市面上的AI编程工具，不管装机量多大，核心逻辑都一样：**它们是给程序员用的。** 它们假设你懂编程，帮你写得更快。

AutoCode 走了一条完全不同的路：

|  | 目前的AI编程工具 | **AutoCode** |
| --- | --- | --- |
| **面向谁** | 程序员 | **所有人** |
| **前提条件** | 你得懂编程 | **你会说话就行** |
| **核心逻辑** | 帮你写更好的代码 | **替你把产品做出来** |
| **技术决策** | 问你选什么语言、什么框架 | **自动选最优方案，用省钱/快/稳来解释** |
| **记忆** | 多数没有，少数要装数据库 | **自动记忆，零配置** |
| **开发费用** | 不优化 | **降低60%+** |
| **学你习惯** | 不学 | **越用越懂你** |
| **情绪反馈** | 冰冷的技术输出 | **进度、鼓励、效率对标、每日总结** |
| **学习曲线** | 需要学习专业术语和方法论 | **零** |
| **依赖** | 从轻到重不等 | **零** |

**AutoCode 不是一个"更好的"AI编程工具。它是一个完全不同的东西。**

你用它不是为了"编程更快"。你用它是因为你根本不需要编程——你只要说出你要什么。
