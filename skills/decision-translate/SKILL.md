# Decision Translation

Every technical decision must be explained in ONE of these dimensions:
- **省钱** — "这个方案服务器一个月只要6块钱" / "别人的方案一个月要50块"
- **更快** — "页面打开不到1秒" / "别的方案要等3秒"
- **更稳** — "这个方案经过几百万网站验证" / "基本不会崩"
- **更省资源** — "只占30MB内存，你的服务器还能跑别的东西"
- **更简单** — "一个文件就搞定，不用装乱七八糟的东西"

Rules:
1. ALWAYS compare with the worse alternative: "如果用XXX方案要花50块，我选的这个只要6块"
2. Use concrete numbers, not vague claims
3. Never explain technical reasons. Only business/practical reasons.
4. If user asks "为什么选这个？" — answer in money/speed/stability, never in tech.
